116 ; For processor 0, Routine Description:
117 ;
118 ; This routine is called when the NT system begins execution.
119 ; Its function is to initialize system hardware state, call the
120 ; kernel initialization routine, and then fall into code that
121 ; represents the idle thread for all processors.
122 ;
123 ; Entry state created by the boot loader:
124 ; A short-form IDT (0-1f) exists and is active.
125 ; A complete GDT is set up and loaded.
126 ; A complete TSS is set up and loaded.
127 ; Page map is set up with minimal start pages loaded

129 ; physical memory.

591 ;++
592 ;
593 ; Routine Description:
594 ;
595 ; This routine is called during kernel initialization once for each
596 ; processor. It sets EM+TS+MP whether we are emulating or not.
597 ;
598 ; If the 387 hardware exists, EM+TS+MP will all be cleared on the
599 ; first trap 07. Thereafter, EM will never be seen for this thread.
600 ; MP+TS will only be set when an error is detected (via IRQ 13), and
601 ; it will be cleared by the trap 07 that will occur on the next FP
602 ; instruction.
603 ;
604 ; If we're emulating, EM+TS+MP are all always set to ensure that all
605 ; FP instructions trap to the emulator (the trap 07 handler is edited
606 ; to point to the emulator, rather than KiTrap07).
607 ;
608 ; Arguments:
609 ;
610 ; None.
611 ;
612 ; Return Value:
613 ;
614 ; None.
